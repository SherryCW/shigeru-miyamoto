## 2. 已知立场测试（Sanity Check）

独立测试三个宫本有公开记录的问题：

| 问题 | Skill 输出方向 | 公开材料对照 | 判定 |
|---|---|---|---|
| 游戏是否应先从故事出发 | 先验证核心行动；故事放大体验而非遮盖玩法；承认强作者叙事例外 | TIME 2017、Pikmin/Wii Music/早期 Mario 与 Zelda 复盘 | **一致** |
| 新硬件是否应追求最高规格 | 规格服务新动作、响应和输入；不把参数领先本身当价值 | Nintendo FY2020/2024、Wii 与 HD Rumble 访谈 | **一致** |
| 核心不好玩时是否继续加内容 | 冻结新增内容，回到最小循环和可玩原型 | Punch-Out!!、Pikmin 单页流程与开发复盘 | **一致** |

结论：**3/3 方向一致，无明显立场偏离。**

### 测试后修正

- 把“先别加故事”改为“核心行动未成立时，暂停用故事遮盖”，防止漂移成“反故事”。
- 在 `dokuso` 局限中明确：不反对前沿技术，反对的是把规格领先本身当玩家价值。
- 把“禁止加内容”改成阶段规则：核心循环通过最小试玩后可恢复内容生产。

## 3. 边缘测试（Edge Case）

**问题：**在单人 RPG 中，是否应把生成式 AI 实时 NPC 对话作为核心玩法？

测试输出没有声称宫本的真实立场，而是基于动作—反馈、原型认识论、自然因果和玩家发现模型提出：

- 一个房间、三个 NPC、十五分钟事件的最小原型；
- 固定分支对话作为对照组；
- 玩家动词为提问/诱导、判断证词、采取行动、观察世界结果；
- 六项失败判据，包括事实冲突、无行动后果、无法形成判断、固定版本表现相同或更好；
- 若移除生成技术后学习—判断—行动循环不变，则它只是内容生产/表现工具，不是核心玩法。

| 检查项 | 结果 |
|---|---|
| 标注基于哪些模型推断 | PASS |
| 避免冒充本人真实立场 | PASS |
| 最小原型与失败判据 | PASS |
| 模型失效条件 | PASS |
| 未越界到电影/IP/商业 | PASS |

结论：**5/5 通过。**

### 测试后修正

- 对本人未公开讨论的新技术，强制写明“基于模型推断，不代表真实立场”。
- 原型最低规格增加：假设、场景、动词、对照、指标、失败判据和失败后的降级动作。
- 增加“核心玩法 vs. 表现/生产工具”的移除测试。

## 4. 表达测试（Voice Check）

**测试场景：**一个开放世界游戏用三分钟过场一次解释多套系统。

测试回答采用“重写问题 → 新玩家试玩 → 发现只追图标 → 改成推箱/开门/惊动敌人的小房间 → 玩家裁决”的结构。

| 维度 | 评分（5分） |
|---|---:|
| 具体场景与动作 | 5.0 |
| 起初—试作—发现—修改因果 | 5.0 |
| 强判断与软语气 | 4.5 |
| 我—团队—玩家主语分工 | 5.0 |
| 自嘲/轻幽默自然度 | 4.0 |
| 无通用 AI 鸡汤味 | 4.0 |
| 不过度模仿 | 4.5 |

总体辨识度：**8.8/10，高。** 删除名字后仍能从具体动作、微型试玩、团队异议与玩家裁决中识别宫本式结构。

### 测试后修正
